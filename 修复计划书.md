## 二、P0 紧急（安全硬伤 + 即刻生效的性能快修）

### 0.1 认证默认关闭

| 维度 | 内容 |
|------|------|
| **问题** | `config.py:78` — `auth_enabled: bool = False`，所有 API 端点默认裸露 |
| **来源** | 安全审计 C1、代码审查 C5 |
| **攻击场景** | 任何网络可达的客户端可创建/删除/执行项目，无需认证 |
| **修复方案** | 默认值改为 `True`；本地开发通过 `.env` 设置 `BLUEPRINT_AUTH_ENABLED=false` |
| **影响范围** | `config.py`、`.env.example`、README |
| **工时** | 5min |
| **依赖** | 无 |

### 0.2 CORS 通配 + 凭证

| 维度 | 内容 |
|------|------|
| **问题** | `main.py:28-30` — `allow_origins=["*"]` + `allow_credentials=True` |
| **来源** | 安全审计 C3、代码审查 C2 |
| **攻击场景** | 任意恶意网站可冒充登录用户发起跨域请求 |
| **修复方案** | `allow_origins` 从 `config.py` 读取，默认 `["http://localhost:5173"]` |
| **影响范围** | `main.py`、`config.py` |
| **工时** | 15min |
| **依赖** | 无 |

### 0.3 所有 API 端点无认证

## 七、讨论决策点

### 决策 1：认证策略

当前 `auth_enabled=False` 是为了本地开发方便。

**建议方案**：保留开关，但默认值改为 `True`；本地开发通过 `.env` 设置 `BLUEPRINT_AUTH_ENABLED=false`。

### 决策 2：CORS 配置

当前 `allow_origins=["*"]`，生产环境必须收紧。

**建议方案**：从 `config.py` 读取允许的 origin 列表，默认 `["http://localhost:5173"]`，生产环境通过环境变量配置。

### 决策 3：修复顺序

**建议方案**：按 P0 → P1 → P2 → P3 顺序执行，每个优先级内并行处理无依赖的任务。

### 决策 4：MemorySaver → SqliteSaver（1.5）

这是最大的架构变更，影响状态持久化逻辑。

**建议方案**：放在 P1 早期执行，因为后续多 worker、水平扩展都依赖它。

### 决策 5：项目所有权（1.3）

需要改数据库 schema + 所有 API 加过滤，工作量 3h，影响面大。

**建议方案**：本迭代完成，因为安全审计明确标记为 HIGH。
