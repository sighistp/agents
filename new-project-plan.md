# AI 开发团队 — 多 Agent 协同系统设计初稿

> **日期：** 2026-06-03
> **状态：** 初稿，待讨论
> **目标：** 构建一个多 Agent 协同的 AI 开发团队，用户输入需求，Agent 自动完成架构设计、编码、测试、审查

---

## 2. 系统架构

### Agent 角色设计

| 角色 | 职责 | 输入 | 输出 |
|------|------|------|------|
| **产品经理** | 需求分析、任务拆解 | 用户原始需求 | 用户故事、功能清单、优先级 |
| **架构师** | 技术方案设计 | 用户故事 | 技术架构、模块划分、接口定义、数据模型 |
| **程序员** | 编码实现 | 架构设计 + 接口定义 | 可运行的代码 |
| **测试员** | 编写测试 + 执行测试 | 代码 + 需求 | 测试用例、测试报告、Bug 列表 |
| **审查员** | 代码审查 | 代码 + 测试报告 | 审查意见、改进建议 |
| **项目经理** | 协调整体流程 | 所有 Agent 输出 | 进度报告、最终交付物 |

### 工作流程

```
用户输入需求
    ↓
产品经理 → 需求分析 → 用户故事 + 功能清单
    ↓
架构师 → 技术方案 → 架构图 + 接口定义 + 数据模型
    ↓
程序员 → 编码实现 → 源代码文件
    ↓
测试员 → 编写测试 → 测试用例 + 执行结果
    ↓
审查员 → 代码审查 → 审查报告 + 修改建议
    ↓
程序员 → 根据审查修改代码（循环）
    ↓
项目经理 → 汇总 → 最终交付物（代码 + 文档 + 测试报告）
```

### 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| Agent 框架 | **LangGraph** | 图结构精确控制流程，支持循环和条件分支，状态持久化 |
| LLM | DeepSeek v4 | 成本低，中文好，已有经验 |
| 前端 | 原生 HTML/CSS/JS | 与 RAGv3 一致，展示底层能力 |
| 后端 | FastAPI | 与 RAGv3 一致 |
| 工具 | 代码执行沙箱、文件读写、测试运行 | Agent 需要实际操作代码 |
| 存储 | SQLite | 项目历史、Agent 对话记录 |

## 3. 核心功能

### 3.1 需求分析
- 用户输入自然语言需求（如"做一个待办事项 Web 应用"）
- 产品经理 Agent 分析需求，输出：
  - 用户故事列表
  - 功能清单（优先级排序）
  - 技术约束

### 3.2 架构设计
- 架构师 Agent 根据用户故事设计：
  - 技术架构（前端/后端/数据库选型）
  - 模块划分（目录结构）
  - API 接口定义
  - 数据模型

### 3.3 编码实现
- 程序员 Agent 根据架构设计编写代码：
  - 生成完整项目文件
  - 支持多种语言（Python/JS/HTML）
  - 代码风格遵循最佳实践

### 3.4 测试
- 测试员 Agent：
  - 根据需求编写测试用例
  - 在沙箱中执行测试
  - 输出测试报告（通过/失败/覆盖率）

### 3.5 代码审查
- 审查员 Agent：
  - 检查代码质量
  - 发现潜在 Bug
  - 提出改进建议
  - 如果有问题，反馈给程序员修改（循环）

### 3.6 可视化流程
- 前端展示 Agent 协作过程：
  - 每个 Agent 的思考过程
  - Agent 间的对话
  - 当前进度
  - 最终交付物

## 4. 与 RAGv3 的关系

| 维度 | RAGv3 | AI 开发团队 |
|------|-------|------------|
| 定位 | 知识库问答 | 代码生成 |
| Agent | 单 Agent + 4 工具 | 多 Agent 协同 |
| 前端 | 暗黑科技风聊天 | 流程可视化 + 聊天 |
| 共享 | FastAPI、SQLite、JWT 认证 | 可复用后端架构 |

**可以复用的模块：**
- `rag/auth.py` — JWT 认证
- `rag/user_db.py` — 用户系统
- `rag/memory.py` — 对话记忆
- `rag/resilience.py` — 重试/熔断
- `rag/guard.py` — 安全防护
- 前端暗黑科技风 CSS 变量

## 5. 差异化（vs MetaGPT/ChatDev）

| 维度 | MetaGPT/ChatDev | 我们的方案 |
|------|----------------|-----------|
| 部署 | 需要本地安装 | Web 应用，浏览器直接用 |
| 可视化 | 命令行输出 | 实时可视化 Agent 协作过程 |
| 交互 | 一次性生成 | 支持人工介入确认 |
| 复用 | 独立项目 | 复用 RAGv3 后端架构 |
| 成本 | 依赖 OpenAI | 用 DeepSeek，成本低 10 倍 |
